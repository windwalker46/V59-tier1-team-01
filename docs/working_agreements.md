# Working Agreements

## Communication
- Daily standups posted in team channel
- Major decisions discussed in team threads

## Meetings
- Sprint planning meetings scheduled as needed
- Async coordination preferred

## Git Workflow
- main branch = production
- feature branches for work
- PR required before merge

## Commits
- Clear, descriptive messages

## Code Reviews
- At least one teammate review when available

## Deployment
- Deploy from main branch
